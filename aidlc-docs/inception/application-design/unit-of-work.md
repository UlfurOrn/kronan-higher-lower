# Units of Work
# Krónan Higher or Lower Game

---

## Unit 1: Data Pipeline

### Identity
- **Unit Name**: data-pipeline
- **Type**: Standalone Node.js/TypeScript CLI script
- **Entry Point**: `scripts/fetch-data.ts`
- **Run Command**: `npm run fetch-data`
- **Runtime**: Node.js LTS (build-time / developer machine only)

### Purpose
Queries the Krónan Public REST API and produces a static `products.json` file that the React application consumes at runtime. This unit is never deployed — it runs locally whenever product data needs refreshing.

### Responsibilities
- Authenticate with the Krónan API using `KRONAN_API_TOKEN` from the environment
- Fetch all products across all categories (with pagination)
- Compute price-per-unit for each product
- Filter to only products with: an image URL, a non-zero price, and a computable price-per-unit
- Write the eligible products as `GameProduct[]` to `src/data/products.json`
- Print a pipeline summary (totals and exclusion reasons) to stdout

### Components Owned
- `PipelineOrchestrator`
- `KronanApiClient`
- `PricePerUnitCalculator`
- `ProductFilter`
- `ProductDatasetWriter`

### Output Artifact
- `src/data/products.json` — array of `GameProduct` objects; this is the contract with Unit 2

### Code Organisation
```
scripts/
└── fetch-data.ts          <- Entry point (calls PipelineOrchestrator)

src/
├── pipeline/
│   ├── KronanApiClient.ts
│   ├── PricePerUnitCalculator.ts
│   ├── ProductFilter.ts
│   └── ProductDatasetWriter.ts
└── types/
    └── index.ts           <- Shared types (GameProduct, KronanRawProduct, etc.)
```

### Definition of Done
- [ ] Script runs to completion with a valid `KRONAN_API_TOKEN`
- [ ] `products.json` contains only products with image + price + price-per-unit
- [ ] Pipeline summary printed to stdout with correct counts
- [ ] Rate limit respected (no 429 errors)
- [ ] Unit tests pass for `PricePerUnitCalculator` and `ProductFilter`

## Unit 2: React Game Application

### Identity
- **Unit Name**: react-game-app
- **Type**: React + TypeScript Single Page Application
- **Entry Point**: `src/main.tsx`
- **Dev Command**: `npm run dev`
- **Build Command**: `npm run build`
- **Runtime**: Browser (served by Vite dev server locally)

### Purpose
The playable Higher or Lower game. Presents two Krónan grocery products and asks the player to guess which has the higher price per unit. Supports Normal (3 lives) and Hard (1 life) modes, streak tracking in localStorage, and an Icelandic UI throughout.

### Responsibilities
- Load and parse `src/data/products.json` at startup
- Present a setup screen for mode and category selection
- Run the game loop: show product pairs, accept answers, reveal prices, advance rounds
- Manage lives, streak, and best streak state
- Show the Game Over screen with final stats and replay options
- Persist best streak to localStorage

### Components Owned
- `App`
- `GameStateProvider` (Context + Reducer)
- `gameReducer`
- `GameLogicService`
- `ProductPoolService`
- `LocalStorageService`
- `SetupScreen`
- `CategoryPicker`
- `GameScreen`
- `HUD`
- `ProductCard`
- `ProductImage`
- `AnswerButtons`
- `GameOverScreen`

### Definition of Done
- [ ] App loads without errors with a valid `products.json` in place
- [ ] Setup screen renders with mode and category selection working
- [ ] Game loop runs: correct answers advance the round, wrong answers cost a life
- [ ] Normal mode starts with 3 lives, Hard mode with 1 life
- [ ] Streak increments on correct answers, resets on wrong answer
- [ ] Best streak persisted to and read from localStorage
- [ ] Game Over screen shows final streak, best streak, and losing round
- [ ] All UI text is in Icelandic
- [ ] Product images render with fallback placeholder on error
- [ ] Unit tests pass for `gameReducer` and `GameLogicService`

## Development Sequence

Unit 1 must produce a `products.json` before Unit 2 can be fully tested with real data.
During initial Unit 2 development, a small sample `products.json` (5–10 hand-crafted records) should be committed to unblock frontend work.

| Step | Action |
|---|---|
| 1 | Develop and test Unit 1 (data pipeline) |
| 2 | Run pipeline to generate real `products.json` |
| 3 | Develop and test Unit 2 (React game app) using real data |
